OpenStack IPV6地址简介

OpenStack 网络支持租户网路中使用IPV6,这就可以为租户内的虚拟机动态的分配IPV6地址.

其中OpenStack 网络也可以集成物理路由器上的SLAAC,从而使虚拟机可以从以后的dhcp系统中获取IPV6地址.

IPV6的子网可以存在的租户网络中创建,并支持以下的地址分配模式,SLAAC,Stateful DHCPv6, Stateless DHCPv6,

IPV6地址获取的三种模式

  • SLAAC(Stateless Adress Autoconfiguration) 利用路由宣告信息RA来确定前缀和长度,利用EUI-64 算法接口算出接口ID,该种分配地址的方法 不需要DHCPv6 server. 运行IPV6的路由器可以为寻找IPv6的客户端提供网络前缀以及网关地址,IPV6使用NDP(邻居发现协议),使用RS 路由请求,和RA路由通告信息,帮助IPV6客户端设备自动配置IPV6地址,无状态地址的好处是在没有dhcp的网络中无需手工配置IP地址,省事省力,网络的整体迁移比较容易,当一个站点的网络前缀发生变化时,主机能够方便地进行重新编址而不影响网络链接.

    OpenStack 中SLAAC分为两种,

    1
2

    第一种为 利用OpenStack router路由宣告消息RA 报文来生成IPV6地址
第二种为 利用外部router路由宣告消息RA报文来 生成IPV6地址
    • 客户端的IPV6地址基于RA(Router advertisement)
    • 网关也同样被配置
    • 可选, DNS信息,MTU 域名等信息
    • 仅支持/64 的前缀信息

  • DHCPv6 statefull 有状态的IPV6地址配置,DHCPv6是一种地址配置协议,它可以获得IP地址, IP前缀 和IPV6网络中所需要操作的其他配置数据, DNS,域名 等

  • DHCPv6 stateless. 无状态的IPV6地址配置,一般是 客户端由RA信息来获取自己的地址配置信息,其他的可选信息等 从dhcpv6处获得

OpenStack中IPV6模式

  • ipv6_ra_mode 主要是定义 谁 来发布RA信息, 用于控制子网的RA发布。

  • ipv6-address-mode, 定义虚拟机怎么包含IPV6地址,默认的网管.和可选信息,

在OpenStack中 使用这两种属性,必须要求enable_dhcp必须为true

SLAAC
ipv6_ra_mode ipv6_address_mode Result
SLAAC 不指定 地址由neutron路由器指定
不指定 SLAAC 地址用EUI-64,外部路由器用户提供路由
SLAAC SLAAC 地址用EUI-64,OpenStack 虚拟路由器用户提供路由，这里提供的RADVD由L3namespace中运行的radvd进程管理

设置ipv6_ra_mode =slaac时, 这时,将由OpenStack路由器将被配置发送RA数据包信息,,在被创建该网络并挂载到虚拟机路由后,将设置地址的配置flags在RA信息中,

  • Auto Configuration Flag = 1
  • Managed COnfiguratioN Flag = 0
  • Other Configuration Flag = 0
DHCPv6

对于DHCPv6-stateless. 目前支持包含的方式如下

ipv6_ra_mode ipv6_address_mode Result
DHCPv6-stateless DHCPv6-stateless 地址由neutron routrerRA分配，可选信息由dhcp 分配
DHCPv6-stateful DHCPv6-stateful 地址以及可选信息全部由dhcpv6提供

ipv6_ra_mode 由 DHCPv6-stateless提供，neutron 路由器会使用radvd agent 发送RA信息，其中附带的标记信息如下

  • Auto Configuration Flag = 1
  • Managed COnfiguratioN Flag = 0
  • Other Configuration Flag = 0

路由支持

在neutron中由于不需要使用浮动IP,所以这里不需要使用L3的路由模式，在整个云平台的使用过程中，目前可以使用的 有两种方式

  1. pd prefix delegation
  2. bgp
