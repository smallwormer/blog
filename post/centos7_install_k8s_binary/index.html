安装方式说明

yum安装

默认在CentOS的的源中可以直接安装k8s，比较简单，但是更新比较麻烦，需要等yum有更新包才可以，并且有些依赖无法自身指定，定制化自身的k8s就比较麻烦，编写spec等等。

二进制安装

二进制安装 好处是可以安装任何版本的kubernetes,直接替换二进制的可执行文件即可，坏处是配置比较复杂，很多软件包的一些配置，都需要自己配置，并且编写服务等。但是比较方便的话安装便捷，可以定制化程度高

kubeadmin安装

Kubeadmin 是Kubernetes官方提供的用于快速安装Kubernetes集群管理工具，伴随Kubernetes版本发布都会同步更新，比较方便快捷，如果只是做些实验能很快的完成安装。使用Kubeadmin安装的Kubernetes是Kubernetes的本身服务容器化的形式提供服务

vim /usr/lib/systemd/system/flanneld.service
[Unit]
Description=Flanneld overlay address etcd agent
After=network.target
After=network-online.target
Wants=network-online.target
After=etcd.service
Before=docker.service

[Service]
Type=notify
EnvironmentFile=/etc/sysconfig/flanneld
EnvironmentFile=-/etc/sysconfig/docker-network
ExecStart=/usr/bin/flanneld \
            $FLANNEL_ETCD_ENDPOINTS \
            $FLANNEL_ETCD_PREFIX \
            $FLANNEL_OPTIONS
ExecStartPost=/usr/local/libexec/mk-docker-opts.sh -d /run/flannel/docker
Restart=on-failure

[Install]
WantedBy=multi-user.target
RequiredBy=docker.service

# etcd config key.  This is the configuration key that flannel queries
# For address range assignment
FLANNEL_ETCD_PREFIX="-etcd-prefix=/atomic.io/network"

# Any additional options that you want to pass
FLANNEL_OPTIONS=""
EOF
