EVE-NG 模拟器网络

说明

在使用eve-ng的使用中 下载的OVA导入到vmware中后，使用的nat的模式，这里遇到了一个问题

虚拟机使用 nat cloud0 方式下 虚拟机 无法获得DHCP，即便是配置了静态IP也无法出网的问题。

问题如下：

拓扑图: cloud0—–vm

网络图具体为： eth0—–linuxbridge cloud0(pnet0)—vm

遇到问题：

  1. 虚拟机 dhcp 不到ip
  2. dhcp配置了ip后 在eve-ng 的虚拟机上 既可以 ping通到该虚拟机，有可以ping通到 母机
  3. 抓包问题为 iou known 发现母机的vmnet8 已经分配了IP, 已经回包，但是无法给虚拟机配置IP
  4. 已经开启了内核的ip转发

变相解决

  1. 配置eth1的 桥 pnet1 (etc/network/interface) 配置静态ip 192.168.145.1/24

    iface eth0 inet manual

# Cloud devices
iface eth1 inet manual
auto pnet1
iface pnet1 inet static
    address 192.168.145.1
    netmask 255.255.255.0
    bridge_ports eth1
    bridge_stp off

  2. 开启iptables 使其nat到 eth0 的桥上 变相出网

    1

    iptables -t nat -A POSTROUTING -o pnet0 -s 192.168.145.0/24 -j MASQUERADE
